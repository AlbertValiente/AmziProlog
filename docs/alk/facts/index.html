Facts and Terms

Prolog comes from the same theoretical roots as relational databases. A Prolog logic base is composed of facts and rules.

Facts in Prolog are similar in form to rows of a relational database table. Unlike a database, Prolog facts are stored as text in a Prolog source file. They can be indexed, sorted or compiled for faster runtime access.

Here are some examples of facts from an application for a sample airline, Amzi! Air:

    airport('I68', 'Lebanon, OH').
airport('WDR', 'Winder, GA').
airport('SPG', 'St. Petersburg, FL').
airport('M22', 'Russellville, AL').
airport('1A0', 'Soddy Daisy, TN').

They relate airport identifiers with the names of the towns the airport services.

Here are some facts about Amzi! Air flight numbers and departure and arrival airports:

    
flight(aa01, 'I68', 'WDR').
flight(aa03, 'WDR', 'SPG').
flight(aa05, 'SPG', 'M22').

Prolog Jargon

A Prolog fact is of the form: 
    predicate_name(argument_1, ..., argument_N).

A predicate is a collection of facts, each having the same predicate_name and the same number of arguments. It corresponds to a table in a relational database.

The arity of a predicate is the number of arguments. This corresponds to the number of columns in a relational table.

A predicate is often represented by predicate/arity. So the two examples above are airport/2 and flight/3.

A fact is more formally called a clause in Prolog. So above we show three clauses of flight/3.

Prolog Term Types

Unlike a database, Prolog facts are typeless. The arguments can be any type of term allowed in Prolog. Two common types are atoms and numbers.

Atoms contain text that is stored internally in an atom table, allowing for very fast pattern matching as predicates are searched. Some atoms from the above examples are:

    
'I68'
'Lebanon, OH'
aa01

Any characters (ASCII, multibyte or wide Unicode) enclosed in single quotes are atoms. The quotes are not necessary if the atom is composed of all letters and the initial letter is lower case.

Numbers are integers or decimals or floating point. Here are some facts about base airfares in the Amzi! Air logic base:

    
base_fare(aa03, 100).
base_fare(aa04, 100).
base_fare(aa05, 90).
base_fare(aa06, 90).

In this base_fare/2 predicate, the second argument is an integer representing the fare.

For the examples, it makes sense that each argument holds the same type of term, but that is not necessary in Prolog, as will be seen later on in the tutorial.

NOTE ==> In actual applications, factual knowledge as shown here might be stored in a Prolog logic base, or might be accessed from external data stores. Either approach works well with Amzi! as will be described later in the tutorial.

Prolog Text

The facts are stored in text files. They can be edited using any program editor. The Amzi! IDE contains an editor.

You can open the file alk/facts/amzi_air.pro to see more of the facts that will be used in the Amzi! Air sample application.
